# 코드 리뷰 체크리스트

> 모든 코드 변경에 대한 리뷰 기준

---

## 1. 테스트 검증

- [ ] 새 코드에 대한 테스트가 존재하는가?
- [ ] 모든 테스트가 통과하는가?
- [ ] 테스트 커버리지가 유지 또는 개선되었는가?
- [ ] 엣지 케이스가 테스트되었는가?
- [ ] skip/pending 테스트가 추가되지 않았는가?

---

## 2. 코드 품질

### 가독성

- [ ] 변수/함수 이름이 의도를 명확히 표현하는가?
- [ ] 복잡한 로직에 주석이 있는가?
- [ ] 함수가 단일 책임을 가지는가?
- [ ] 중첩 깊이가 3단계 이하인가?

### 유지보수성

- [ ] 중복 코드가 없는가?
- [ ] 매직 넘버/문자열이 상수로 정의되었는가?
- [ ] 의존성이 명확한가?
- [ ] 확장하기 쉬운 구조인가?

---

## 3. 보안

- [ ] 사용자 입력이 검증되는가?
- [ ] 민감 정보가 로그에 노출되지 않는가?
- [ ] SQL 인젝션, XSS 등 OWASP Top 10 취약점이 없는가?
- [ ] 하드코딩된 비밀키/토큰이 없는가?

---

## 4. 성능

- [ ] 불필요한 루프/반복이 없는가?
- [ ] N+1 쿼리 문제가 없는가?
- [ ] 대용량 데이터 처리 시 메모리 고려가 되었는가?
- [ ] 캐싱이 적절히 사용되었는가?

---

## 5. 문서화

- [ ] 공개 API에 문서가 있는가?
- [ ] 복잡한 알고리즘에 설명이 있는가?
- [ ] CHANGELOG 업데이트가 필요한가?
- [ ] README 업데이트가 필요한가?

---

## 6. t-ruby 특화 항목

### 컴파일러 변경

- [ ] 새 문법이 spec 문서에 반영되었는가?
- [ ] RBS 출력이 올바른가?
- [ ] 타입 에러 메시지가 명확한가?
- [ ] 기존 .trb 파일과 호환되는가?

### 에디터 플러그인 변경

- [ ] 모든 에디터(VSCode, JetBrains, Vim, Neovim)에 동일 기능이 적용되었는가?
- [ ] LSP 프로토콜을 준수하는가?
- [ ] 버전 동기화가 필요한가?

### 문서 변경

- [ ] 코드 예제가 실제로 컴파일되는가?
- [ ] 3개 언어(영어, 한국어, 일본어) 번역이 동기화되었는가?

---

## 7. 커밋 메시지

- [ ] Conventional Commits 형식을 따르는가?
- [ ] 변경 이유가 명확한가?
- [ ] 관련 이슈가 참조되었는가?

### 형식 예시

```
feat(compiler): 맵드 타입 지원 추가

- MapType AST 노드 구현
- RBS 출력에서 맵드 타입 변환
- 관련 테스트 15개 추가

Closes #123
```

---

## 8. 승인 기준

### 필수 통과

- 모든 테스트 통과
- 보안 취약점 없음
- 코드 스타일 (RuboCop) 통과

### 권장 통과

- 커버리지 유지/개선
- 문서 업데이트 완료
- 성능 영향 검토 완료
