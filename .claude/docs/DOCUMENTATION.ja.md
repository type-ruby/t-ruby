# ドキュメントガイドライン

> このドキュメントはt-rubyプロジェクトのドキュメント標準を定義します。

**翻訳:** [English](./DOCUMENTATION.md) | [한국어](./DOCUMENTATION.ko.md)

---

## 1. ドキュメントの種類

### サービス用ドキュメント

ソフトウェアユーザー向けのドキュメント。

- **目的:** インストールガイド、使用方法チュートリアル、APIリファレンス
- **場所:** プロジェクトルート（`README.md`など）
- **主言語:** 英語（翻訳版は同一内容を維持）

### 開発用ドキュメント

プロジェクト貢献者向けのドキュメント。

- **目的:** アーキテクチャ仕様、コーディング標準、内部API
- **場所:** `.claude/`ディレクトリ
- **主言語:** 英語（翻訳版は同一内容を維持）

---

## 2. 言語ポリシー

### 標準言語

**英語**がプロジェクトの標準言語です。

### 必須翻訳

すべてのドキュメントは3言語で維持される必要があります：

| 言語 | サフィックス | 例 |
|------|-------------|-----|
| 英語 | (なし) | `README.md` |
| 日本語 | `.ja` | `README.ja.md` |
| 韓国語 | `.ko` | `README.ko.md` |

### 翻訳リンク

各ドキュメントの上部に翻訳版へのリンクを含める必要があります：

```markdown
**翻訳:** [English](./README.md) | [한국어](./README.ko.md)
```

## 3. ドキュメント簡素化原則

### 行数制限

すべての**開発用ドキュメント**は**200行未満**である必要があります。

### 宣言

すべての開発用ドキュメントは上部に準拠宣言を含める必要があります：

```markdown
<!-- Follows Document Simplification Principle: max 200 lines -->
```

### 大容量ドキュメントの分割

ドキュメントが200行を超える場合：

1. ドキュメントと同名のフォルダを作成
2. 重いセクションから順に別ファイルに分離
3. メインファイルは概要と分割セクションへのリンクのみを維持
4. すべての分割ファイルも200行制限を遵守

**例：**

分割前：
```
.claude/
└── spec.md (350行)
```

分割後：
```
.claude/spec/
├── README.md (概要、〜80行)
├── syntax.md (分割セクション)
├── type-system.md (分割セクション)
└── ast.md (分割セクション)
```

## 4. ファイル命名規則

### サービス用ドキュメント

```
README.md           # 英語（デフォルト）
README.ja.md        # 日本語
README.ko.md        # 韓国語
```

### 開発用ドキュメント

```
.claude/docs/DOCUMENTATION.md      # このガイド
.claude/spec/README.md             # 言語仕様
```

### 分割ドキュメント

フォルダに分割する際は`README.md`をメインエントリポイントとして使用：

```
.claude/{トピック}/
├── README.md        # 概要 + 目次
├── section-a.md     # 分割セクション
└── section-b.md     # 分割セクション
```

---

## 5. メンテナンスルール

1. **翻訳の同期:** ドキュメント修正時はすべての翻訳版も同時に修正
2. **行数確認:** ドキュメントが200行未満であることを確認
3. **リンク更新:** 変更後、相互参照が有効であることを確認
4. **宣言確認:** すべての開発用ドキュメントに簡素化原則宣言が含まれていることを確認
