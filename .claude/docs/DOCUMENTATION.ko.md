# 문서화 가이드라인

> 이 문서는 t-ruby 프로젝트의 문서화 표준을 정의합니다.

**번역:** [English](./DOCUMENTATION.md) | [日本語](./DOCUMENTATION.ja.md)

---

## 1. 문서 종류

### 서비스용 문서

소프트웨어 사용자를 위한 문서.

- **목적:** 설치 가이드, 사용법 튜토리얼, API 레퍼런스
- **위치:** 프로젝트 루트 (`README.md` 등)
- **주 언어:** 영어 (번역본은 동일하게 유지)

### 개발용 문서

프로젝트 기여자를 위한 문서.

- **목적:** 아키텍처 사양, 코딩 표준, 내부 API
- **위치:** `.claude/` 디렉토리
- **주 언어:** 영어 (번역본은 동일하게 유지)

---

## 2. 언어 정책

### 표준 언어

**영어**가 프로젝트의 표준 언어입니다.

### 필수 번역

모든 문서는 3개 언어로 유지되어야 합니다:

| 언어 | 접미사 | 예시 |
|------|--------|------|
| 영어 | (없음) | `README.md` |
| 일본어 | `.ja` | `README.ja.md` |
| 한국어 | `.ko` | `README.ko.md` |

### 번역 링크

각 문서 상단에 번역본 링크를 포함해야 합니다:

```markdown
**번역:** [English](./README.md) | [日本語](./README.ja.md)
```

## 3. 문서 간소화 원칙

### 줄 수 제한

모든 **개발용 문서**는 **200줄 미만**이어야 합니다.

### 선언

모든 개발용 문서는 상단에 준수 선언을 포함해야 합니다:

```markdown
<!-- Follows Document Simplification Principle: max 200 lines -->
```

### 대용량 문서 분기

문서가 200줄을 초과할 경우:

1. 문서와 동명의 폴더를 생성
2. 무거운 섹션부터 순서대로 별도 파일로 분리
3. 메인 파일은 개요와 분기된 섹션 링크만 유지
4. 모든 분기 파일도 200줄 제한 준수

**예시:**

분기 전:
```
.claude/
└── spec.md (350줄)
```

분기 후:
```
.claude/spec/
├── README.md (개요, ~80줄)
├── syntax.md (분기된 섹션)
├── type-system.md (분기된 섹션)
└── ast.md (분기된 섹션)
```

## 4. 파일 명명 규칙

### 서비스용 문서

```
README.md           # 영어 (기본)
README.ja.md        # 일본어
README.ko.md        # 한국어
```

### 개발용 문서

```
.claude/docs/DOCUMENTATION.md      # 이 가이드
.claude/spec/README.md             # 언어 사양
```

### 분기된 문서

폴더로 분기 시 `README.md`를 메인 진입점으로 사용:

```
.claude/{주제}/
├── README.md        # 개요 + 목차
├── section-a.md     # 분기된 섹션
└── section-b.md     # 분기된 섹션
```

## 5. 유지보수 규칙

1. **번역 동기화:** 문서 수정 시 모든 번역본도 함께 수정
2. **줄 수 확인:** 문서가 200줄 미만인지 확인
3. **링크 갱신:** 변경 후 교차 참조가 유효한지 확인
4. **선언 검토:** 모든 개발용 문서에 간소화 원칙 선언 포함 확인
